# Туториал по работе с Git

## Начало работы

Для начала работы, необходимо инициализировать сам Git.

Для его инициализации введите команду: 

```
  git init
```


## Добавление файла

Для добавления файла в Git необходимо воспользоваться командой: 

```
git add название файла
```
## Представление Git-у

Случается так, что Git не имеет представления с кем он работает, поэтому для дальнейшей работы необходимо ему предстваиться.

Для этого используй команду:

```
git config
```

## Отображение состояния рабочего каталога

Для того, чтобы проверит индексацию изменений и увидеть файлы, которые не отслеживаются или отслеживаются в Git, используют следующую команду:

```
git status
```

## Сохранение , зафиксировать текущий статус

После изменений, которые произошли в процессе работы с файлами в Git необходимо зафиксировать все эти изменения.

Для этого следует использовать команду:

```
git commit

Если вы хотите оставить какое-то сообщение используйте ключ -m
git commit -m "Your message"

А также, если вы нехотите после каждого изменения все фиксировать командой git add, вы можете сократить свои действия до одной команды используя ключ -a, или -am/ -a -m, если хотите еще оставить и комментарий
git commit -a
git commit -a -m "Your message"
git commit -am "Your message"
```

## Откат коммита

Допустим, вы поняли, что фиксация, которую вы совершили, была произведена с ошибкой или она в принципе Вам и не очень нужна, для этого следует использовать следующую команду:

```
git rever <номер коммита>
```

## Изменение сообщения последнего коммита

Допустим, вы закоммитили последнию фиксацию с ошибкой или какой-то неточностью. У Вас возникла необходимость переписать сообщение, используйте команду:

```
git commit --amend -m "Your message"
```

## Журнал сохранений

Чтобы посмотреть все фиксации вы можете обратиться к команде:

```
git log
```
Которая выведет все изменения и человека, выполнявшего все эти изменения, на той ветке, на которой вы вписали данныу команду.

## Переход к другой версии, переходит между фиксациями

Допустим, вам необходимо перейти между фиксациями, что-то подправить - изменить, добавить или убрать. Тогда используйте команду:

```
git checkout _ _ _ _ , где нижние подчёркивания - первые четыре символа коммита, в которой Вы хотите перейти, либо Вы можете вставить номер коммита целиком, не имеет значения
```

## Показывает разницу между текущем состоянием файла и файлом сохранённым

Команда, которая выводит все неподтвержденные изменения, внесенные после последнего коммита, выглядит следующим образом:

```
git diff
```

## Отменяет текущую фиксацию и текущий коммит, который ещё не создан или не сохранен, до фиксации

Если Вы еще не создали или не сохранили фиксацию или коммит, вы можете это отменить командой:

```
git stash
```

## Сбрасывает до последнего фиксированного режима (откат)

Ну либо Вы можете просто вернуться в последнию фиксацию командой:

```
git reset
```

## Вся история, которая происходила в файле, все изменения

Если Вам необходимо посмотреть все изменения(всю историю) работы в репозитории, Вы можете ввести команду:

```
git reflog
```
Она покажет Вам все изменения.

## Команда с лог коммитов и визуализацией между ними

Эта команда позволит Вам увидеть все коммиты и что в них происходило:

```
git log --graph 
```

## Список всех имеющихся веток

Посмотреть все ветки, которые у Вас есть, предоставит возможность команда:

```
git branch
```

## Создание веток

Для того, чтобы создать ветку, используйте команды:

```
git branch branch_name
git checkout -b branch_name или git switch -c branch_name , для моментального создания и перехода в эту ветку
```

## Переход между ветками

Чтобы перейти в ту или иную ветку для работы, используйте команды:

```
git checkout branch_name
git switch branch_name
```

## Команда слияния веток

Если Вы произвели какие-то изменения в ветке, например, в черновике и Вам необходимо сделать отображение содержимого этой ветки в основной ветке, следующая команда позволи Вам это сделать:

```
git merge branch_name
```

## Удаление уже слитой ветки 

Допустим, у Вас есть какая-то ветка, и она вообще не нужна используйте:

```
git branch -d branch_name - удалит ветку, сохранив историю с коммитами

git branch -D branch_name - удалит ветку, забрав с собой всю историю работы в мусорный бак.
```

## Переименование веток

 Вам надоело название ветки? XD 
 Ну или Вам надо её переименовать, потому что по ошибке Вы написали там что-то неприличное)))
 Данная команда спасёт Вас:

 ```
git branch -m branch_name(старое название) branch_name(новое название)
 ```

## Клон удаленного репозитория

Вы захотели поработать с чьим-то удалённым репозиторием у себя в локальном, команда:

```
git clone <link...>
```

## Команда выгрузки содержимого локального репозитория в удалённый репозиторий

Вы сделали свою работу и хотите отправить её в удалённый репозиторий, для этого используйте команду:

```
git push
```

## Команда извлечения изменений из удаленного репозитория и объединяет их с вашей локальной веткой

Вы хотите поработать с чьим-то удалённым репозиторием у себя в локальном, действуйте:

```
git pull
```

## Команда, которая извлекает изменения из удаленного репозитория и обновляет ваши локальные указатели ветвей

В заголовке всё написано, а я устал это писать, поэтому команда:

```
git fetch
```

## Показывает список всех удаленных репозиториев, о которых знает Git

Аналогично, всё в заголовке, команда:

```
git remote
```

## Добавляет новый удаленный репозиторий

Ну тут тоже, прочитав заголовок, догоните о чём спич:

```
git remote add
```

## Удаляет существующий удаленный репозиторий

Ну я уже сказал выше, поэтому просто на команду:

```
git remote remove
```

## Переименовывает существующий удаленный репозиторий

Команда:

```
git remote rename 
```

## Показывает информацию о конкретном удаленном репозитории

Команда:

```
git remote show
```

## Создание, удаление и переход к файлам

```
touch name_of_file - создание файла
cd Name/ - переход к файлу
cd .. - переход к файлу выше
git rm(remove) file_name - удаление файла
```

## Конфликты

Когда возникают конфликты при слиянии веток, надо выбрать путь, по которому Вы пойдете. Описывать не буду, Git предложит и покажет всё за меня.
И вдруг, **ДОСАДА**!!! Вам вообще эти изменения не подходят, откатитесь и урегулируйте конфликт между строками пока не началась *кибер-война*. Команда:

```
git merge --abort
```
